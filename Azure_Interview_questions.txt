Q1. What is Cloud?

Ans- Cloud is just another server in a remote location like a home NAS device.

Q2. What is Azure?

Ans- Azure is Microsoft’s Cloud Computing platform. It is a combination of services for creating, deploying, 
and managing applications that run in Microsoft’s secure worldwide data centers.
 Almost any software that can be imagined can be written to take advantage of Azure services and can run in the Microsoft Cloud.

Q3. What is a Private Cloud?

Ans- Private cloud is a cloud that is created by an organization for their own use and their own employees can only access that. Here the datacenter is privately stored in the organization itself.

Q4. Is cloud service a collection of Data Center?

Ans- Cloud Service is not exactly a Data Center but it has the similarity to a data center. And that is why Cloud Service is also referred to as a Remote Data Center.

Q5. How is the Cloud different from Mainframe? It was also hosting applications remotely?

Ans- Yes Cloud also stores and runs applications on remote servers. But, unlike the Mainframe where there is a centralized location to store the Server in Cloud, there is no centralized location to store the server. It is scattered across many individual servers across different geographical locations.

Q6. What is Azure SLA for uptime?

Ans- 99.9 % uptime.

Q7. What is Auto-scaling in Azure?

Ans- The ability to increase and decrease resources when needed. It is generally referred to as Scale-up and Scale down.

Q8. How many Azure Regions are there?
	Microsoft Azure currently consists of 60+ regions worldwide, available in 140 countries.

Q9. Difference between ASM vs ARM?
	ASM (Azure Service Manager ):
	ASM stands for Azure Service Management. This is usually termed as Classic Deployment model. 
	The classic resources can be provisioned and managed using Azure Classic portal, 
	which can be reached at https://manage.windowsazure.com.

	ARM ( Azure Resource Manager):  
	ARM stands for Azure Resource Manager. This deployment model of Azure represents a different 
	way to deploy and manage your Azure resources than in ASM model. Azure ARM portal can be reached 
	at https://portal.azure.com.

Q10. Network Security Group (Firewall) : 
	A network security group contains security rules that allow or deny inbound network traffic to, or 
	outbound network traffic from, several types of Azure resources
	
Q11. Router Table: Azure automatically routes traffic between Azure subnets, virtual networks, and on-premises networks. 
	If you want to change any of Azure's default routing, you do so by creating a route table.

Q14. Azure Firewall: Azure Firewall is a managed cloud-based network security service that protects your Azure Virtual Network resources.

Q17. How to create Virtual Machine:
	Virtual Machines in All Services
	Click on Add icon
	Select Subscription and Resource Group
	Enter the Virtual Machine Name
	Image: Windows Server 2016 datacenter
	Size: changes size and select b2ms
	Enter user name and password
	Public inbound ports: None
	Click on Newdisk: Select Standard SSD
	Click on Next Networking
	Select Virtual Networking: AzureClassvNet1
	Select Subnet: AzureClassSubnet1
	Network security Group: Basic
	Public Inbound Group: None
	Click Next Management
	Enable Basic plan for free: No
	Boot Diagnostics: off
	OS Guest Diagnostics: off
	System Assigned auto Identity: off
	Enable Auto  Shutdown: off
	Enable Backup: off
	Click on Advanced
	Click on Tags : Env - Prod
	Click review + create
	Click Create
	
Q18. Allow RDP Port (3389) in Virtual Machine:

	Visit Virtual Machine
	Click on Networking
	Click on Network security group: myfirstmachine-nsg
	Click on inbound security rules
	Click on Add icon
	Source: Any
	Destination: Any
	Destination Port ranges:3389 (RDP:Remote Desktop port)
	Priority: 100
