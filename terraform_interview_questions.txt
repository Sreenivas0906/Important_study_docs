15)Define null resource in Terraform.
In Terraform, a null resource is a placeholder resource that doesn’t create any real infrastructure. 
It’s used to perform tasks like running scripts or managing dependencies between resources using provisioners or triggers.
---->Triggers:
You can use the triggers argument to define when a null resource should be recreated. These triggers can depend on the output or state of other resources.
----->Local Execution:
It can work with provisioners (like local-exec or remote-exec) to run commands or scripts locally or on a remote machine.
----->Dependency Management:
Helps manage complex dependencies between resources without creating physical infrastructure.

19) What is State File Locking?
--->State file locking in Terraform is a mechanism to prevent multiple users or processes from making simultaneous changes to the same Terraform state file. It ensures that changes to the infrastructure state are serialized, avoiding conflicts or corruption of the state file.
Key Features:
Locking Mechanism: Terraform locks the state file during operations (e.g., apply, refresh) to ensure only one operation modifies it at a time.
Remote State Backends: Locking is supported by remote backends like S3 (with DynamoDB), Azure Blob Storage, and Terraform Cloud, which handle the locking automatically.
Unlocking: The lock is released once the operation is complete. If an operation is interrupted, manual unlocking might be required using the terraform force-unlock command.
Prevents Conflicts:By locking the state file, Terraform prevents issues like overwriting or inconsistent state updates when multiple users or CI/CD pipelines are involved.

20)What is a Remote Backend in Terraform?
----In Terraform, a remote backend in Azure refers to using Azure Storage as the backend to store the Terraform state file remotely. 
Centralized State File: The Terraform state file is stored in an Azure Blob Storage container, enabling teams to access it from multiple locations.
State Locking: By integrating with Azure's blob lease mechanism, Terraform ensures state file locking, preventing simultaneous operations from corrupting the state.
Versioning: AZURE Storage supports blob versioning, which allows rollback to a previous state version if needed.
Security: State files are stored securely with options for encryption (e.g., server-side encryption with Azure Key Vault).

32)What are the various levels of Sentinel enforcement?
Sentinel has three levels of enforcement: advisory, soft mandatory, and hard mandatory.
----- >Advisory – Logged in but permitted to pass. When a user initiates a plan that violates the policy, an advisory is issued.
------>Soft Mandatory – Unless an override is specified, the policy must be followed. Overrides are only available to administrators.
------>Hard Mandatory – The policy must be implemented regardless. Unless and until this policy is removed, it cannot be overridden. Terraform’s default enforcement level is this.

37) What is the benefit of Terraform State?
----->The Terraform State is a critical feature in Terraform that serves as a record of the infrastructure resources managed by Terraform. It contains mappings between the resources defined in your configuration and the actual infrastructure that exists. Here are the key benefits of using Terraform State:
----->Resource Management:
The state file helps Terraform track the current configuration of your infrastructure and its dependencies, ensuring that it knows the state of each resource it manages. This enables Terraform to plan changes and identify differences between the desired and actual state.
---->Efficient Change Detection:
By maintaining the state file, Terraform can compute a minimal set of changes required to transition your infrastructure from the current state to the desired state (as defined in your .tf configuration files). This helps to optimize resource creation, modification, and deletion, avoiding unnecessary changes.
---->Collaboration:
When using remote backends like AWS S3, Azure Blob Storage, or Terraform Cloud, the state file can be shared among team members. This ensures that everyone works from a consistent view of the infrastructure, avoiding conflicts or inconsistencies when changes are applied.
----->Metadata and Outputs:
The state file stores metadata and outputs that are essential for resource interdependencies. For instance, the output values from one resource (like an instance IP address) can be used as inputs for another resource (like setting up a security group).
---->Rollback and Recovery:
If a deployment fails or something goes wrong, the state file can be used for recovery. Since it maintains an up-to-date record of your infrastructure, Terraform can help you roll back to the last known good state or restore resources to their previous configuration.
---->Terraform Cloud Integration:
When using Terraform Cloud, state management is fully automated and stored in a centralized, secure manner. Terraform Cloud also provides features like versioning, access controls, and automatic locking, which ensures better security and collaboration for teams managing infrastructure as code.

39) How will you upgrade plugins on Terraform?
----> Run ‘terraform init’ with ‘-upgrade’ option. This command rechecks the releases.hashicorp.com to find new acceptable provider versions. 
It also downloads available provider versions. “.terraform/plugins/<OS>_<ARCH>” is the automatic downloads directory.
