Large Language Model (LLM)::
=============================
           * LLMs are the LANGUAGE MODELS made up of NEURAL NETWORK with BILLIONS OF PARAMETERS that are trained by SELF-SUPERVISED LEARNING on vast amounts of UNLABLED TEXT..

HOW DO LLM WORKS ::
=====================

1. Training Phase::
---------------------
a. Data Collection.           
           *LLMs are trained on massive datasets like books, articles, websites, and other text sources.
b. Tokenization (or) preprocessing. 
           *Text is broken into smaller units called tokens (e.g., words or subwords) for easier processing.
c. Neural Network Architecture (or) model architecture.
           *LLMs, often based on the Transformer architecture, process tokens in parallel, capturing relationships and context between words using mechanisms like self-attention.
d. Learning Patterns (or) Learning .
           *The model learns patterns in language by predicting the next token in a sequence, optimizing itself to minimize prediction errors.

2. Inference (Using the Model)
-------------------------------
a. Input Processing
           *User inputs are tokenized and fed into the model.
b. Context Understanding
           *The model uses its learned knowledge to analyze the context of the input.
c. Output Generation
           *The model predicts and generates the most likely next tokens, forming coherent and relevant responses or outputs

BASICS OF PROMPT ENGINEERING::
==============================
             * prompt engineering is as much an art as it is a science .
 prompt engineering consists of 2 words . that are
           Prompt -----> A detailed set of guidelines given to an LLM to do a task .
           Engineering ----> Developing a task-specific prompt iteratively. 

      *** Prompt Engineering is an Iterative Process

COMPONENTS OF A GOOD PROMPT ::
==============================
              * Clarity: Clearly state the task or question without ambiguity.
              * Context: Provide enough background information to ensure understanding.
              * Specificity: Define the scope, desired output, or format explicitly.
              * Conciseness: Use precise language to avoid unnecessary complexity.
              * Goal Orientation: Focus on the outcome you want to achieve.
              * Relevance: Ensure the prompt aligns with the user's or system's capabilities.
              * Engagement: Use language that encourages creativity or critical thinking.
              * Examples (if needed): Provide examples or templates to guide the response.
              * Constraints: Define any boundaries or restrictions for the task.
              * Open-endedness (if appropriate): Allow flexibility for creative or comprehensive responses.

 Flask GPT app
================

   - Flask is a lightweight Python web framework used to build web applications quickly and easily.

2. What is GPT?  
   - GPT (Generative Pre-trained Transformer) is a language model that can generate human-like text based on user input.

3. Flask GPT App Purpose  
   - It combines Flask and GPT to create an application where users can interact with GPT (e.g., ask questions or generate content).

4. How it Works  
   - Flask handles the web interface (frontend) and routes requests to the backend.
   - GPT processes user inputs on the backend and generates responses.

5. Frontend
   - The user interacts with the app via a web page (HTML/CSS/JavaScript).

6. Backend  
   - Flask receives user inputs, sends them to GPT for processing, and returns the responses to the frontend.

7. API Integration  
   - The app often uses APIs (e.g., OpenAI API) to connect with GPT for generating responses.

8. Use Cases 
   - Chatbots, virtual assistants, content generation tools, coding assistants, and more.

9. Easy to Deploy  
   - A Flask GPT app can be deployed on platforms like Replit, Heroku, or any cloud service.

10. Why Use Flask for GPT?x  
    - Simple to set up, flexible, and ideal for small to medium-sized applications.

Retrieval- Augmented Generation (RAG)::
========================================
RAG (Retrieval-Augmented Generation) is a framework for improving AI model responses by combining information retrieval with language generation. Here's a clear breakdown:

Key Components RAG :
---------------------
         * Retrieval: Finding relevant information from a knowledge base
         * Augmentation: Adding this retrieved information to the prompt
         * Generation: Creating responses using both the retrieved context and the model's training

How it works RAG :
-------------------
         * User asks a question
         * System searches through stored documents/data for relevant information
         * Retrieved information is added to the prompt
         * AI model generates response using both its training and the retrieved context

Main Benefits RAG :
--------------------
         * More accurate and up-to-date responses
         * Reduced hallucinations by grounding answers in specific sources
         * Ability to access information beyond the model's training data
         * Better handling of domain-specific questions
         * Real-time access to updated information

Common Use Cases RAG:
---------------------
         * Customer support systems
         * Documentation search
         * Research assistance
         * Knowledge base querying
         * Technical support
         * Internal company tools
BASIC WORKFLOW OF RAG ::
---------------------------

          * User Query: The user asks a question or provides input.
          * Search for Data: The system retrieves relevant information from a database or documents.
          * Combine with Query: The retrieved data is added to the user’s query for more context.
          * Generate Response: A language model (like GPT) uses the query and data to create a response.
          * Return Answer: The system sends the enriched, accurate response back to the user.

Steps to implement RAG with langchain ::
-----------------------------------------
 1) Install required packages------- > pip install langchain chromadb openai
 2) Import necessary modules:
                 * from langchain.embeddings import OpenAIEmbeddings
                 * from langchain.vectorstores import Chroma
                 * from langchain.text_splitter import CharacterTextSplitter
                 * from langchain.chains import ConversationalRetrievalChain
                 * from langchain.chat_models import ChatOpenAI

 3) Load your document(s):
                 * with open('your_document.txt', 'r') as file:
                     * documents = file.read()


 4) Split text into chunks:
                 * text_splitter = CharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
                 * texts = text_splitter.split_text(documents)

 5) Create embeddings using OpenAI:
                  * embeddings = OpenAIEmbeddings(openai_api_key='your-api-key')
           
 6) Create vector store:
                 * vectorstore = Chroma.from_texts(texts, embeddings)

 7) Initialize the retrieval chain:
                 * qa_chain = ConversationalRetrievalChain.from_llm(
                        llm=ChatOpenAI(),
                        retriever=vectorstore.as_retriever()
                    )

 8) Query the system: 
                 * response = qa_chain({"question": "your question", "chat_history": []})
                   print(response['answer'])
